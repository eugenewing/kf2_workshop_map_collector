# NEXT_STEPS

## Цели следующего этапа

- [x] Создание файла-схемы для описания эвристики классификации карт (взвешенные сигналы), чтобы детально видеть, как работает логика определения карт.

- [x] Создание ещё двух JSON-файлов для карт:
  - архивные карты,
  - забагованные карты.
  Карты в эти файлы должны помещаться вручную через фронт.
  Далее `maps.json`, `review.json` и эти два файла считаются **четырьмя списками карт**.

- [x] Реализация возможности поиска карты по её `id` в каждом из четырёх списков.

- [x] Реализация функционала ручного удаления карт из четырёх списков и их перемещения между четырьмя списками на фронте.
  Для каждого элемента в списке забагованных карт должно быть обязательно заполнено поле `причина`.
  При перемещении карты в этот список фронт должен обязательно запрашивать у пользователя причину помещения карты в список забагованных.

- [x] Добавление кнопки быстрого копирования `id` карты в буфер обмена для каждого элемента четырёх списков на фронте.

- [x] Добавление пятого списка карт `ignored.json` для элементов, которые были ошибочно добавлены программой в списки `maps.json` и `review.json` и на самом деле не являются картами. Данный список предназначен для ручного добавления в него карт. Для него, также как и для других списков карт, должен быть реализован стандартный функционал: возможность поиска элемента по его `id`, ручное удаление элементов, ручное перемещение элементов из данного списка в другие и из других списков в данный список, а также кнопка быстрого копирования `id` элемента. Для `ignored.json`, также как и для других списков карт, должно быть справедливо правило обязательного указания причины добавления элемента в список `bugged.json`.

- [x] Добавление шестого списка карт `featured.json` для элементов, которые были однозначно определены пользователем как карты. Данный список предназначен для ручного добавления в него карт. Для него, также как и для других списков карт, должен быть реализован стандартный функционал: возможность поиска элемента по его `id`, ручное удаление элементов, ручное перемещение элементов из данного списка в другие и из других списков в данный список, а также кнопка быстрого копирования `id` элемента. Для `featured.json`, также как и для других списков карт, должно быть справедливо правило обязательного указания причины добавления элемента в список `bugged.json`. Таким образом на данном этапе будет всего шесть списков карт. `maps.json` и `review.json` будут заполняться программой и считаться в дальнейшем **двумя основными списками**. `archived.json`, `bugged.json`, `ignored.json` и `featured.json` же будут заполняться пользователем на фронте и считаться в дальнейшем **четырьмя пользовательскими списками**.

- [x] Установка элементов всех выпадающих списков для перемещения карт по кнопке "Move" в другие списки по следующей последовательности: `featured.json`, `archived.json`, `ignored.json`, `bugged.json`, `maps.json`, `review.json`.


- [x] Реализация логики сверки найденных элементов во время парсинга с **четырьмя пользовательскими списками**. Перед добавлением найденного элемента в один из **основных списков** программа должна проверить его наличие в **четырех пользовательских списках** и, в случае его обнаружения в них, игнорировать этот элемент во время парсинга.

- [ ] Добавление функции блокирования кнопок перемещения во всех шести списках во время обновления **основных списков** по кнопке Refresh.

- [ ] Поддержка загрузки картинок для каждой карты в шести списках:
  - отображение изображений карты на фронте,
  - хранение массивов ссылок на изображения для каждой карты в шести JSON-списках,
  - кнопки быстрого копирования ссылки на изображение в буфер обмена для каждого изображения.

- [ ] Реализация функционала создания, редактирования и удаления тегов для карт на фронте.
  Список всех тегов должен сохраняться в отдельном JSON в папке data.
  Элемент тега должен иметь три поля:
  - id (уникальный id тега в пределах данного JSON, должен генерироваться автоматически при создании тега, не должен редактироваться),
  - обязательное название,
  - опциональное описание.

- [ ] Реализация функционала добавления тега к карте прямо на фронте для списка отобранных карт (`featured.json`). Массив добавленных к карте тегов должен сохраняться в элементе карты в `featured.json` в специальное поле в виде списка id тегов и отображаться на фронте в списке отобранных карт под названием карты в виде названий тегов.

- [ ] Реализация функционала удаления специальных полей элементов в списках при их перемещении. Два json списка из шести имеют поддержку специальных полей под каждый элемент. У `featured.json` это опциональное поле со списком тегов для карты, у `bugged.json` это обязательное поле с описанием причины добавления карты в данный список. При перемещении карты из этих двух списков в любые другие четыре списка данные специальные поля карты должны удаляться.

- [ ] Реализация функционала сортировки всех карт по добавленным к ним вручную тегам в списке `featured.json` на фронте.
